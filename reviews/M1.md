## Architect Decisions

### Decision 1 — Reconciled schema shape (resolves Blocked #1)

Implement the **reconciled shape now**; do not implement a literal `05` schema:

1. **Dimension vocabulary.** The canonical dimension enum is the superset:
   `correctness | reliability | security | performance | efficiency | cost | reproducibility | maintainability`.
   `maintainability` is valid for `Finding.dimension` only (per `05`); it is not scored in the MVP.
2. **ScoreCard.dimensions is a mapping, not fixed fields:** `dimensions: dict[DimensionName, DimensionScore]` where `DimensionScore = {score: float (0–100), confidence: float (0–1)}`. Which dimensions are present and how they are weighted is decided by the scoring policy, not the schema. The MVP policy `agent_mvp_default` (`10_REPORTING_AND_SCORING.md`) uses correctness, reliability, security, performance, efficiency, reproducibility; `cost` remains schema-valid for findings, metrics, and future policies. This satisfies both `05` and `10` without a breaking change later.
3. **Confidence:** per-dimension confidence (in `DimensionScore`) **plus** one overall `confidence` on the ScoreCard. `05`'s single number becomes the overall; `10`'s requirement becomes the per-dimension field.
4. **ScenarioResult.status** gains a first-class terminal value: `passed | failed | error | timed_out | skipped`. `06`'s lifecycle requires TimedOut as distinct from error; `05`'s enum was under-specified. A timed-out scenario is `timed_out`, never `error`.
5. Document this reconciliation as **ADR-006: Reconciled scoring dimensions and scenario status** (short; cite the two conflicting specs and this review). Spec files themselves must not be edited.

### Decision 2 — `gauntlet init` layout (resolves Blocked #2)

Codex's proposed layout is **authorized** as the public contract:

```text
<project>/
├── .gauntlet/
│   ├── config.yaml              # minimal valid config per 11_CLI (project, adapter, evaluation, execution, reporting, scoring)
│   ├── profiles/default.yaml    # sample profile
│   ├── benchmarks/README.md     # placeholder explaining where packs go (keeps dir in git)
│   └── adapters/python_callable.py
└── .gauntletignore
```

- **Adapter template:** a provisional stub, not comment-only — a module with a documented `run(payload: dict) -> dict` function whose body raises `NotImplementedError`, plus comments showing the intended shape and noting the contract is finalized in Milestone 2 (ADR-002/ADR-003). It must import nothing from `gauntlet` internals so it stays valid when M2 lands.
- **Overwrite behavior:** `gauntlet init` is idempotent and non-destructive — creates missing files, never overwrites existing ones, reports what it created vs skipped; `--force` overwrites. Exit code 0 on success including no-op.

### Decision 3 — Default artifact root (resolves Blocked #3)

Codex's proposed resolution is **authorized**:

- Default artifact root is **`~/.gauntlet/artifacts`** (user home, expanded cross-platform via `Path.home()`), with the canonical ADR-001 layout `runs/<run_id>/` beneath it. Never default to `<project>/artifacts` — `09_SECURITY_AND_THREAT_MODEL.md` requires outputs outside project source, and the ADR-001/`04` `artifacts/` path describes the GAUNTLET development repo, not user projects.
- The root is overridable via the standard precedence chain (CLI flag > env var `GAUNTLET_ARTIFACT_ROOT` > project config `artifacts.root` > profile > package default). Tests and CI use the override.
- Run manifests must include `project_id` so `runs list` can distinguish projects sharing the root.
- Document as **ADR-005: Artifact root outside project source** (short).
