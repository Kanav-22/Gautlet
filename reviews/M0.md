## Blocked Item — Architect Decision

**Question raised by Codex:** the plan's M0 gate reads `pip install -e . && gauntlet --version && pytest`, but `pytest`/`ruff`/`mypy` are classified as development dependencies, so a literal fresh venv install provides no `pytest` command.

**Decision: Option 1 — the gate's install step is `pip install -e ".[dev]"`.** `pytest`, `ruff`, and `mypy` remain in an optional `dev` dependency group (`[project.optional-dependencies] dev = [...]` in `pyproject.toml`). Runtime users must never receive test tooling from a plain `pip install`. The plan's gate wording was shorthand for "install the project with its dev tooling in a fresh venv"; this review is the authoritative clarification. The canonical M0 gate is:

```bash
python -m venv <fresh> && <fresh>/bin/pip install -e ".[dev]"
gauntlet --version
pytest
```

This clarification applies to all subsequent milestone gates that invoke `pytest`, `ruff`, or `mypy`.
